## UC 1.1 Авторизоваться в системе {#uc-1.1-авторизоваться-в-системе}

| Название | Авторизоваться в системе  |
| :---- | :---- |
| Акторы | Работник |
| Триггер | Неавторизованный пользователь зашел на сайт |
| Предусловие | Работник имеет табельный номер и пароль для своей учетной записи. |
| Постусловие | Пользователь авторизован в системе Пользователь может выполнять разрешенные действия в системе в рамках текущей сессии Пользователю отображается главный экран рабочего пространства. |
| Основной сценарий “Успешная авторизация в системе” | Пользователь открывает веб\-интерфейс на экранной форме входа.  Пользователь вводит свой табельный номер и пароль в соответствующие поля формы.  Пользователь выбирает действие “Выполнить вход” Система проверяет соответствие введенного пароля и учетной записи по табельному номеру. При совпадении данных пользователь входит в свою учетную запись.  Система отображает главную страницу  рабочего пространства.  |
| Альтернативный сценарий “Неправильный пароль или табельный номер” | п.1-3 Основного сценария Несоответствие  пароля для данного табельного номера.  Система выводит сообщение с предупреждением     о неправильном вводе табельного номера или пароля.   Вход в учетную запись по данному табельному номеру не происходит.  Пользователю продолжает отображаться  интерфейс входа в систему.  Пользователь может повторить попытку входа еще N раз. (Если после N раза вход не осуществлен, то возможность ввода для данной формы блокируется на M-минут  |
| Альтернативный сценарий  1 “Пользователь восстанавливает пароль” | Пользователь открывает веб\-интерфейс на экранной форме входа.  Пользователь выбирает действие “Восстановить пароль” Система отправляет ссылку для смены пароля на почту, привязанную к данной учетной записи. Пользователь переходит по ссылке.  Система отображает поле для смены текущего пароля:  1\.  Ввод нового пароля  2\.  Повторный ввод пароля для подтверждения Также отображаются действия:  1\. Сменить пароль Действие “Сменить пароль” не доступно без ввода данных в оба поля. Пользователь вводит новый пароль в оба поля.  Пользователь выбирает действие сменить пароль. Система проверяет соответствие двух введенных строк с паролем. Если строки совпадают система прерывает все остальные активные сеансы для данной учетной записи и изменяет связанный с ней пароль на новый.  Пользователь может вернуться на экранную форму входа и выполнить вход с новыми данными. |
| Альтернативный сценарий “Пользователь не может восстановить пароль” | п.1-8 Альтернативного сценария 1 Если данные полей не совпадают, система выдает предупреждение о несоответствии пароля и подтверждающего пароля и просит проверить ввод. Пользователь меняет содержимое полей, как необходимо. п. 7-10 Альтернативного сценария 1 |

## UC 1.5 Просматривать уведомления {#uc-1.5-просматривать-уведомления}

| Название | Просматривать уведомления |
| :---- | :---- |

| Постусловие | Пользователю отображается окно просмотра уведомлений. |

В окне отображаются непросмотренные уведомления в порядке новизны, а также действие “Посмотреть все уведомления” Пользователь выбирает уведомление из списка. Отображается содержимое данного уведомления и действие “Отметить прочитанным” Пользователь выбирает действие “Отметить прочитанное”  Система отмечает уведомление прочитанным и убирает из списка новых уведомлений. |
| Альтернативный сценарий 1  “Просмотр всех уведомлений с отмечанием новых”  | п.1-3 Основного сценария Пользователь выбирает действие “Посмотреть все уведомления”   Пользователю отображается страница просмотра всех уведомлений с действием “Вернуться назад” Новые уведомления отмечаются символом. Пользователь выбирает новое уведомление из списка. Отображается содержимое данного уведомления и действие “Отметить прочитанным” Пользователь выбирает действие “Отметить прочитанное”  Система отмечает уведомление прочитанным и убирает из списка новых уведомлений и снимает символьную отметку. Пользователь может просмотреть любое уведомление неограниченное количество раз. Пользователь выбирает действие “Вернуться назад” Пользователю отображается главная страница рабочего пространства. |
| Альтернативный сценарий 1  “Просмотр всех уведомлений без отметки о новизне” | п. 1-4 Альтернативного сценария 1 Пользователь выбирает не новое уведомление из списка. Отображается содержимое данного уведомления. Пользователь может просмотреть любое уведомление неограниченное количество раз. Пользователь выбирает действие “Вернуться назад” Пользователю отображается главная страница рабочего пространства. |

## UC 2.4 Работать с тестом  {#uc-2.4-работать-с-тестом}

| Название | Работать с тестом |
| :---- | :---- |

| Предусловие | Пользователь авторизован в своей учетной записи. Пользователю отображаются рабочее пространство и страница прохождения темы. |
| Постусловие | Пользователю отображается страница работы с тестом. Содержимое страницы зависит от предыдущих прохождений. |

Пользователь может запросить персональную рекомендацию ИИ-ассистента по повторению темы на основании результата прохождения. Если результат прохождения теста больше 80 процентов, то пользователь не может выбрать действие “Пройти тест заново”. |
| Альтернативный сценарий  “Нет теста больше 80 процентов” | п.1-4 Альтернативного сценария 1 Если результат прохождения теста меньше 80 процентов, то пользователь может выбрать действие “Пройти тест заново”. |
